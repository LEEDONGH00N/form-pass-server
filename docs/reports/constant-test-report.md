## 4. Stages 테스트와 비교

| 항목 | Stages (100 VU peak) | Constant (50 VU) | 차이 |
|------|---------------------|------------------|------|
| 평균 응답 (avg) | 706ms | **236ms** | 3배 개선 |
| p(95) | 1,000ms | **567ms** | 1.8배 개선 |
| 최대 응답 (max) | 4,000ms | **657ms** | 6배 개선 |
| RPS | 53/s | **67/s** | 26% 증가 |
| 성공률 | 100% | 100% | 동일 |

### 분석

- **50 VU**: 안정적인 성능 유지
- **100 VU**: 병목 발생, 응답 시간 급증
- **병목 지점**: 50~100 VU 사이

## 5. 결론

### 5.1 성능 요약

| 항목 | 결과 |
|------|------|
| **최적 동시 사용자** | 50명 |
| **안정 RPS** | 67 req/s |
| **평균 응답 시간** | 236ms |
| **권장 p(95) 기준** | 600ms |

### 5.2 성능 기준 충족 여부

| 기준 | 결과 |
|------|------|
| p(95) < 500ms | ❌ 미충족 (567ms) |
| p(95) < 600ms | ✅ 충족 |
| 에러율 < 1% | ✅ 충족 (0%) |
| 성공률 100% | ✅ 충족 |

### 5.3 권장 사항

1. **p(95) 기준 조정**: 500ms → 600ms
2. **최대 동시 사용자 제한**: 50명 권장, 100명 한계
3. **스케일링 필요 시점**: 동시 사용자 50명 초과 예상 시
