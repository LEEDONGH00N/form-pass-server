# PRD-001: 이벤트 예약 시스템

> 작성일: 2026-03-08
> 상태: Released (v1.0)
> 작성자: 이동훈

## 1. 개요

누구나 사업자 인증 없이 이벤트를 생성하고, 정원 제한이 있는 예약을 받을 수 있는 시스템을 구축한다.

## 2. 배경 및 문제

- 기존 예약 플랫폼(네이버 예약 등)은 사업자 인증과 월정액이 필요
- 구글 폼은 무료지만 재고 관리, 중복 예약 방지가 불가능
- 소규모 행사(동아리, 원데이 클래스, 커뮤니티 모임) 주최자는 간편한 예약 도구가 없음

## 3. 목표

- 회원가입 후 즉시 이벤트를 생성하고 예약 폼을 공유할 수 있는 프로세스 구축
- 정해진 정원만큼만 정확하게 예약을 처리하여 초과 예약 방지
- QR 코드 기반 현장 체크인으로 입장 관리 간소화

## 4. 비목표

- 결제 시스템 연동 (v1.0 범위 아님)
- 게스트 회원가입 (비회원 예약만 지원)
- 대기열 시스템 (별도 PRD로 분리)

## 5. 사용자 및 역할

| 역할 | 설명 |
|---|---|
| 호스트 | 이벤트를 생성·관리하고, 예약 현황을 확인하며, 현장 체크인을 처리하는 주최자 |
| 게스트 | 이벤트 페이지에 접속하여 예약하고, QR 코드를 받아 현장에서 입장하는 방문자 |

## 6. 유스케이스

### UC-1: 호스트 회원가입 및 로그인

**선행 조건**: 없음

```
1. 호스트가 이메일로 회원가입 요청
2. 시스템이 인증 코드를 이메일로 발송
3. 호스트가 인증 코드를 입력하여 이메일 인증 완료
4. 호스트가 이메일, 비밀번호, 이름으로 회원가입
5. 시스템이 JWT 토큰을 발급하여 로그인 완료
```

**예외 흐름**:
- 이미 가입된 이메일 → 에러 반환
- 인증 코드 불일치 → 에러 반환

---

### UC-2: 이벤트 생성

**선행 조건**: 호스트 로그인 상태

```
1. 호스트가 이벤트 제목, 장소, 설명을 입력
2. 호스트가 1개 이상의 스케줄(시작시간, 종료시간, 정원)을 추가
3. 호스트가 커스텀 폼 질문을 추가 (선택사항)
4. 호스트가 이벤트 이미지를 업로드 (선택사항)
5. 시스템이 고유 이벤트 코드를 생성하고 이벤트를 저장
6. 호스트가 이벤트 코드 기반 URL을 게스트에게 공유
```

---

### UC-3: 게스트 예약

**선행 조건**: 이벤트가 공개 상태이고 잔여 좌석이 있음

```
1. 게스트가 이벤트 코드로 이벤트 페이지에 접속
2. 게스트가 원하는 스케줄(시간대)을 선택
3. 게스트가 이름, 전화번호, 티켓 수를 입력
4. 게스트가 커스텀 폼 질문에 답변 (필수 질문이 있는 경우)
5. 시스템이 잔여 좌석을 확인하고 예약을 확정
6. 시스템이 UUID 기반 QR 코드를 발급
7. 게스트가 QR 코드를 확인
```

**예외 흐름**:
- 잔여 좌석 부족 → 에러 반환
- 동일 전화번호로 같은 스케줄 중복 예약 → 에러 반환

---

### UC-4: 호스트 대시보드 및 예약 관리

**선행 조건**: 호스트 로그인 상태, 본인 이벤트

```
1. 호스트가 대시보드에서 총 좌석 수, 예약 수, 체크인 수, 예약률을 확인
2. 호스트가 스케줄별 예약 목록을 조회 (검색, 페이징 지원)
3. 호스트가 개별 예약의 상세 정보(폼 답변 포함)를 확인
4. 호스트가 필요 시 예약을 취소 처리
```

---

### UC-5: 현장 체크인

**선행 조건**: 예약이 확정 상태(CONFIRMED)

```
A. QR 체크인
   1. 게스트가 QR 코드를 제시
   2. 호스트가 QR 코드를 스캔
   3. 시스템이 예약을 확인하고 체크인 처리

B. 수동 체크인
   1. 호스트가 예약 목록에서 게스트를 검색
   2. 호스트가 수동 체크인 버튼을 클릭
   3. 시스템이 체크인 처리
```

**예외 흐름**:
- 이미 체크인된 예약 → 에러 반환
- 취소된 예약 → 에러 반환

---

### UC-6: 예약 조회 (게스트)

**선행 조건**: 없음

```
1. 게스트가 이름과 전화번호를 입력
2. 시스템이 해당 정보로 확정된 예약 목록을 반환
3. 게스트가 예약 상세 및 QR 코드를 확인
```

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 동시성 | 동일 스케줄에 동시 예약 시 정원을 초과하지 않아야 한다 |
| 보안 | 게스트 전화번호는 AES-256으로 암호화하여 저장한다 |
| 보안 | 호스트 비밀번호는 BCrypt로 해싱하여 저장한다 |
| 인증 | JWT Bearer Token (30분 만료, HS256) |
| 권한 | 호스트는 본인이 생성한 이벤트만 관리할 수 있다 |

## 8. 성공 지표

- 동시 예약 환경에서 초과 예약 0건 (방어율 100%)
- 50 VU 기준 평균 응답시간 500ms 이하
- 호스트가 회원가입 후 5분 이내에 이벤트를 생성하고 공유 가능
